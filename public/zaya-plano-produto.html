Visão do Produto

⚡

O que é a Zaya?

Zaya é uma assistente pessoal de IA completa que combina voz, chat, automação, gerenciamento de redes sociais e ligações em uma única solução. Diferente de qualquer outro produto no mercado, a Zaya é instalada diretamente no computador do usuário, garantindo privacidade total dos dados.

Diferencial Competitivo

  • Não é SaaS na nuvem — é instalável no computador do usuário, garantindo privacidade total sobre todas as APIs, dados e conversas
  • Assistente completa — voz, chat, WhatsApp, Instagram, Facebook, ligações, geração de mídia, agendamentos e CRM em um só lugar
  • Setup guiado — a própria Zaya conduz o usuário pelo processo de configuração, sem necessidade de conhecimento técnico
  • Modular — o usuário ativa apenas os módulos que precisa, pagando pelo plano correspondente

Modelo de Negócio

Venda por licença única (pagamento único) com verificação online via token. Cada compra gera um token exclusivo vinculado à máquina do usuário. Após a primeira validação, o sistema funciona 100% offline. Opcionalmente, o usuário pode assinar um plano mensal de suporte e atualizações.

Modelo de Distribuição e Segurança

A Instalador Nativo

Aplicativo Electron que empacota tudo: Node.js, Chrome embarcado, dashboard local. O usuário baixa um .dmg (Mac) ou .exe (Windows), instala e abre — sem terminal, sem configuração manual.

Vantagens

  • Fácil para o usuário final
  • Aparência profissional
  • Sem necessidade de terminal
  • Auto-update integrado

Desvantagens

  • Mais complexo de desenvolver
  • Tamanho grande do instalador
  • Manutenção de duas plataformas

B CLI via Terminal

Instalação via comando: npx install-zaya ou curl -s https://zaya.ai/install | bash. Instala dependências automaticamente e configura o .env de forma interativa no terminal.

Vantagens

  • Instalador leve
  • Rápido de desenvolver
  • Ideal para devs

Desvantagens

  • Exige conhecimento técnico
  • Menos profissional
  • Assusta usuário leigo

C Híbrido (Melhor dos Dois Mundos)

Instalador que abre um terminal embutido junto com o dashboard web. A própria Zaya guia o setup via chat no dashboard — o usuário não precisa digitar comandos, apenas seguir as instruções visuais.

Vantagens

  • Acessível para todos
  • Setup assistido pela IA
  • Flexível e transparente

Desvantagens

  • Complexidade intermediária
  • UX pode confundir

Sistema de Licença / Token

Mecanismo de proteção e validação da compra:

  • Cada compra gera um token único (UUID + hash criptográfico)
  • Na primeira execução, Zaya valida o token online via API de verificação
  • Token vinculado a 1 máquina (fingerprint: MAC address + hostname)
  • Servidor de licenças: endpoint simples que valida token + fingerprint
  • Após validação inicial, funciona 100% offline
  • Planos disponíveis: Básico Pro Enterprise

Bloqueio por Máquina (Anti-Pirataria)

O token é travado no computador onde foi ativado. Se copiar para outro PC, não funciona.

Hardware Fingerprint

Combina múltiplos identificadores únicos da máquina para gerar um hash impossível de replicar:

  • MAC Address — placa de rede (único por hardware)
  • Serial do HD/SSD — número de série do disco
  • UUID da Máquina — identificador do sistema
  • CPU ID — identificador do processador
  • Hostname — nome do computador

Esses 5 dados são combinados em um hash SHA-256. Mesmo clonando o disco, o hash muda em outro hardware.

Fluxo de Ativação
1 Usuário compra → recebe token por email
2 Instala e cola o token na Zaya
3 Zaya gera fingerprint do hardware
4 Envia token + fingerprint → servidor valida
✓ Servidor grava: token X = fingerprint Y
✗ Se outro PC tentar usar o mesmo token → BLOQUEADO
Cenários de Proteção
Cenário Resultado
Usar no mesmo PC✓ Funciona
Copiar pasta para outro PC✗ Bloqueado
Clonar HD para outro PC✗ Bloqueado
Compartilhar token✗ Já ativado
Máquina virtual✗ Detectado
Formatar e reinstalar (mesmo PC)✓ Funciona (mesmo hardware)
Trocar de PC (pedir transferência)⟳ Via suporte (1x/ano)

Implementação técnica: Usar a lib node-machine-id (npm) que gera um UUID único da máquina baseado no hardware. No servidor, uma tabela simples: tokens(token, fingerprint, activated_at, plan). Na inicialização, a Zaya compara o fingerprint local com o registrado no servidor. Se diferente → bloqueia.

Arquitetura Técnica

┌─────────────────────────────────────────────┐ │ ZAYA (Electron App) │ ├──────────────┬──────────────┬────────────────┤ │ Frontend │ Backend │ Services │ │ (HTML/JS) │ (Express) │ │ │ │ │ • AI Engine │ │ Dashboard │ API │ • WhatsApp │ │ Meta Dash │ Routes │ • Voice/TTS │ │ Chat UI │ WebSocket │ • Scheduler │ │ │ │ • Media Gen │ │ │ │ • IG Scraper │ ├──────────────┴──────────────┴────────────────┤ │ SQLite + Supabase (opcional) │ └──────────────────────────────────────────────┘

Camada Frontend

Dashboard HTML/JS puro servido localmente. Inclui interface de chat, painel de métricas do Instagram/Meta Ads, calendário de agendamentos e configurações. Comunicação com backend via WebSocket para tempo real.

Camada Backend

Servidor Express.js rodando em localhost. Expõe API REST para o frontend e gerencia conexões WebSocket. Orquestra todos os serviços (IA, WhatsApp, voz, mídia) e gerencia o estado da aplicação.

Camada de Dados

SQLite como banco local padrão — zero configuração, dados no computador do usuário. Supabase como opção para sync na nuvem, backup e acesso remoto (o usuário escolhe se quer habilitar).

Seção 08

Segurança

Privacidade como pilar fundamental do produto

🔑

API Keys Locais

Todas as chaves de API ficam exclusivamente no computador do usuário. Nunca são enviadas para nenhum servidor externo.

💲

Token Seguro

O token de licença não contém dados sensíveis — apenas um identificador único para validação da compra.

🔒

Validação Mínima

Comunicação com servidor de licença transmite apenas token + fingerprint. Nenhum dado pessoal ou de uso.

💾

Dados Locais

Todas as conversas do WhatsApp, Instagram e dados do CRM ficam armazenados localmente em SQLite.

☁️

Nuvem Opcional

Supabase é totalmente opcional — o usuário escolhe se deseja sincronizar dados na nuvem ou manter tudo local.

🛡

Código Protegido

O código pode ser ofuscado com pkg ou bytenode para proteger a propriedade intelectual.

Seção 09

Capacidades do Sistema Empacotado

Acesso Total ao Computador

A Zaya empacotada com Electron mantém 100% das funcionalidades atuais. O Electron roda Node.js com acesso completo ao sistema operacional — é a mesma tecnologia usada pelo VS Code, Discord, Slack, Figma e Spotify. Não há nenhuma limitação em relação ao que a Zaya faz hoje rodando pelo terminal.

Todas as Funcionalidades Mantidas

🌐
Abrir Navegador & Web Scraping

Puppeteer abre Chrome, navega sites, tira screenshots, faz scraping. Funciona idêntico ao atual — inclusive o scraping de notificações do Instagram.

💬
WhatsApp Completo

Baileys ou WaSender conectam ao WhatsApp sem depender de navegador aberto. Enviar/receber mensagens, áudios, imagens, documentos. Bot automático com IA.

📞
Ligações Telefônicas

Twilio faz ligações reais com voz da Zaya (ElevenLabs). Liga para qualquer número, fala com voz natural, agenda lembretes com ligação automática.

🖥️
Acessar Arquivos do Computador

Node.js fs lê, escreve, organiza arquivos e pastas. A Zaya pode abrir, editar, mover, renomear qualquer arquivo no computador do usuário.

⚡
Executar Comandos no Terminal

child_process.exec executa qualquer comando do sistema. Abrir apps, instalar programas, rodar scripts, automatizar tarefas do OS.

📸
Screenshots & Monitoramento

Captura tela do computador, analisa com IA, monitora o que está acontecendo. Screen monitor em tempo real com análise inteligente.

🎬
Gerar Vídeos & Imagens

Google Veo3, Kling (Freepik), DALL-E, Gemini — gera vídeos e imagens por IA. FFmpeg embutido para edição e conversão de mídia.

📊
Instagram & Facebook Ads

Dashboard completo de Meta Ads com métricas, filtros, análise de IA, scraping de seguidores por anúncio, gestão de campanhas, DMs.

Seção 10

Considerações Finais

Posicionamento de Mercado

A Zaya ocupa um espaço único no mercado: não existe outro produto que ofereça uma assistente de IA completa, instalável no computador do usuário, com privacidade total. Enquanto todos os concorrentes são SaaS na nuvem (onde seus dados ficam nos servidores deles), a Zaya mantém absolutamente tudo local.

Diferenciais Estratégicos

  • Privacidade como produto — no mundo pós-LGPD, privacidade é um argumento de venda poderoso. Nenhuma empresa quer suas conversas e dados de clientes em servidores de terceiros.
  • Pagamento único — o modelo de licença única (não assinatura) é significativamente mais atrativo para o público brasileiro, que resiste a assinaturas mensais recorrentes.
  • Vídeos tutoriais são essenciais — a maior barreira de adoção é a configuração de API keys. Vídeos curtos e claros para cada módulo reduzem drasticamente o volume de suporte.
  • Começar pelo Mac — seu público inicial é Mac-first. Garantir uma experiência impecável no macOS antes de expandir para Windows.
  • Escalar com comunidade — criar um grupo exclusivo de usuários (Telegram/Discord) para feedback, suporte peer-to-peer e evangelismo orgânico.

Próximos Passos Imediatos

  1. Finalizar limpeza do código e sistema de configuração dinâmico
  2. Implementar API de licenças (Supabase Edge Functions)
  3. Criar protótipo do Setup Wizard no dashboard
  4. Empacotar primeiro build com Electron para Mac
  5. Testar com 3-5 usuários alpha antes do beta público
